              About & Methodology
            

            
How Context Bridge metrics are measured and why you can trust them.

          

        


        

          
          

            
            

              

                

                  
                  Token Savings Methodology
                

              

              

                
1. A Fair Baseline

                We do not inflate savings by comparing snippets against your entire repository. We only compare the size of the extracted snippets against the exact size of the specific files retrieved.

                
                
2. Accurate Proportions

                The system uses characters as a highly efficient proxy for tokens. Because we calculate a ratio 1 - (delivered / full), the percentage closely approximates token savings ratios in practice without the overhead of heavy tokenizer libraries.

                
                
3. Real Input Tokens

                The savings shown explicitly represent Input Tokens (prompt tokens), directly reducing LLM API costs and preventing context-window overload.

              

            


            
            

              

                

                  
                  Architecture Overview
                

              

              

                
The Context Bridge retrieval flow typically follows this pipeline:

                

                  query → retrieval → ranking & extraction → filtering → LLM
                

                
This ensures that the final payload delivered to the AI is a compact and highly relevant context.

              

            

          


          
          

            
            

              

                

                  
                  Telemetry & Success Scoring
                

              

              

                
1. Explicit Outcomes (Gold Standard)

                For the highest reliability, metrics are based on actual file usage and task outcomes. AI agents are instructed to report the outcome of a coding task using the record_outcome tool.

                
                
2. Fallback Heuristics

                If an AI fails to report an outcome, the dashboard falls back to local telemetry signals. Low confidence, missing files, and failed tool execution are heuristic indicators, not definitive failures.

                
                
3. Real-World Evaluation

                We evaluate based on real-world confidence and actual file usage. Real-world usage complements benchmark datasets rather than replacing them, providing a dynamic view of how the system solves organic developer problems.

                
                
4. Confidence Scores

                The system generates a confidence score for each query. Confidence reflects retrieval relevance, not answer correctness. It is a probabilistic measure of how well the retrieved code matches the intent of your search.

              

            


            
            

              

                

                  
                  Privacy & Local Operation
                

              

              

                
Context Bridge is designed to operate entirely locally. Search queries, file paths, token statistics, and agent outcomes remain on the user's machine and are not transmitted to external services by Context Bridge itself. No source code bodies, proprietary secrets, or telemetry data are transmitted outside the local environment by Context Bridge itself. Any optional integrations or LLM providers are governed by their own privacy policies and are outside the scope of Context Bridge.
