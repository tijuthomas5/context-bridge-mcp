# Config Before Setup

Before running the setup script for your OS, review:

- `config.json`
- `config.hybrid.json`

## What `config.json` controls

It tells ContextBridge where to look for:

- the central `graphify-out` folder
- optional docs/knowledge folders
- parent folders to scan for nested `graphify-out`
- known ownership `graphify-out` folders

All paths are workspace-relative. Do not use absolute machine paths unless you intentionally want a machine-specific setup.

## If you do not configure it

ContextBridge will still run, but only if your workspace already matches the configured folder structure.

If your project uses different folder names or different locations, setup may:

- miss some Graphify folders
- build an incomplete index
- give weaker search results
- show missing-root or discovery warnings

## Minimum requirement

At minimum, the central root must be correct:

- `graphify-out`

If your repo also has nested ownership graphs, the related discovery roots and ownership roots should also match your workspace layout.

## Setting your source folder names

After setup creates `config.hybrid.json`, open it and find `settings.discovery`.

Replace the placeholder names with your actual project folder names:

```json
"discovery_roots": [
  {"path": "main_service", "kind": "graphify", "role": "ownership_graphify", "match_dir_name": "graphify-out", "required_if_parent_exists": true},
  {"path": "main_ui",      "kind": "graphify", "role": "ownership_graphify", "match_dir_name": "graphify-out", "required_if_parent_exists": true}
],
"ownership_graph_roots": [
  {"path": "main_service/Services/graphify-out", "area": "backend-services", "pack": "secondary-ownership"},
  {"path": "main_ui/src/graphify-out",           "area": "frontend",         "pack": "secondary-ownership"}
],
"source_root_prefixes": ["main_service", "main_ui", "graphify-out", "docs"]
```

These paths are relative to `workspace_root` which defaults to `".."` (one level above `context_bridge/`).

After editing, rerun the setup script for your OS — it is safe to rerun and will rebuild the index with the correct roots.
- **Windows:** `context_bridge\setup\windows\setup_context_bridge.bat`
- **Mac:** `context_bridge/setup/mac/setup_context_bridge.sh`
- **Linux:** `context_bridge/setup/linux/setup_context_bridge.sh`

## Practical rule

- same workspace layout as current config: run setup directly
- different workspace layout: update `config.hybrid.json` first, then run setup
